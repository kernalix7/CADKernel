CADKernel은 초기 개발 단계이므로, 보안 업데이트는 `main` 브랜치의 최신 버전을 우선 대상으로 적용합니다.

## 취약점 제보 방법

**보안 취약점은 공개 GitHub Issue로 제보하지 말아 주세요.**

대신 [GitHub Security Advisories](https://github.com/kernalix7/CADKernel/security/advisories/new)로 제보해 주세요.

### 제보 시 포함할 내용

1. **설명** — 취약점에 대한 명확한 설명
2. **재현 방법** — 상세한 재현 절차
3. **영향도** — 예상 가능한 영향 (예: 데이터 손실, 임의 코드 실행)
4. **영향 범위** — 영향받는 구성요소 (예: 파일 파서, 플러그인 시스템, MCP 서버)
5. **환경 정보** — OS, Rust 버전, CADKernel 버전

### 대응 일정

- **접수 확인** — 제보 후 48시간 이내
- **초기 평가** — 7일 이내
- **수정 및 공개** — 제보자와 조율하여 진행 (치명적 이슈는 통상 30일 이내)

### 범위 (In Scope)

다음 항목은 보안 제보 범위에 포함됩니다:

- 메모리 안전성 이슈 (`unsafe` 블록 관련 포함)
- 파일 포맷 파서 취약점 (악성 입력 처리)
- 플러그인/Add-on 샌드박스 탈출
- MCP 서버 인증/인가 이슈
- 조작된 입력에 의한 서비스 거부(DoS)
- 공급망 보안 이슈 (의존성 취약점)

### 범위 외 (Out of Scope)

- 사용자 장치에 물리적으로 접근해야만 가능한 문제
- 사회공학 기반 공격
- 서드파티 의존성 자체 이슈 (가능하면 upstream에 제보, 필요 시 본 프로젝트에도 공유)

## 보안 모범 사례

CADKernel은 다음 보안 원칙을 따릅니다:

- **`unsafe` 최소화** — 모든 `unsafe` 블록은 문서화 및 리뷰
- **의존성 감사** — `cargo audit`를 통한 정기 취약점 점검
- **퍼징 테스트** — 파일 파서 및 입력 처리에 대한 지속적 퍼징
- **플러그인 격리** — Add-on은 격리된 환경에서 실행

## 감사의 말

책임 있는 공개(Responsible Disclosure)에 기여해 주는 보안 연구자 커뮤니티에 감사드립니다. 유효한 제보자는 동의하에 릴리스 노트에 기여자로 표기할 수 있습니다.
